Arch Linux をインストールした際の備忘録

MacBook Air (2020) 先日、メインPCのOSを Arch Linux にしました。簡単にいけるやろ！と思っていましたが、意外なことに躓いて10時間強を溶かすことになりました。もう二度と繰り返したくないので、備忘録という体でネットに放流します。 インストールディスクの作成 Arch Linux のダウンロードサイトからISOファイルをダウンロードし、ddコマンドで手持ちのUSBに書き込みます。詳しくは↓のリンクを参考にしてください。 USB flash installation medium | Arch Linux https://wiki.archlinux.org/title/USB_flash_installation_medium UEFIのセットアップ 今回のインストールで個人的に最大の鬼門だったのが、UEFIの設定でした。結論から言ってしまえば、Intel VT (VT: Virtualization Technology) なるものがデフォルトで有効化されており、それによってUSBブートができませんでした。(※インストール後に有効化しましたが普通に起動できました) なお、CSM と Secure Boot もデフォルトで有効化されていたのですが、 UEFIを使っている場合はいずれも無効化したほうが良さそうです。 パーティション EFI用、root用、swap用で分割します。ただし、私の環境ではメモリが32GBあるのでswap用のパーティションは作りませんでした。なお、Arch Linux を起動した状態でパーティションを割る場合は当然CUIで行うことになるため、それが怖い方はGPartedなどを他の Live USB 環境で起動1し、GUIで実行したほうが安心できるかもしれません。 いざインストール 'hwha' GParted Live on USB がおすすめです。 ↩︎
